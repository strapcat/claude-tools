The Four Layers

Every Claude Project has four places to put information. Each has a job and a cost. Click each layer to explore.

Think of it like a person showing up to work:
Memory = what they know about you personally · Instructions = their job description · Files = reference binders on the shelf · Notion = the shared company wiki
›
1
Memory
What Claude recalls about you across every conversation
Persistent Short facts Always loaded Max 200 chars each

Put here ✓

  • Your name, role, preferences
  • Claude's role in this project
  • Key Notion IDs & locations
  • Stable preferences & style
  • Cross-project context

Not here ✗

  • Detailed instructions
  • Process steps or workflows
  • Frequently changing info
  • Templates or long content
  • Strategy documents
›
2
Instructions
The system prompt Claude reads before every single message
Most influential Loads every message Keep lean ~1500 words

Put here ✓

  • Role definition & personality
  • Scope & boundaries
  • Behavioral rules (always/never)
  • Output preferences & tone
  • Key workflows (brief)
  • File & Notion references

Not here ✗

  • Long reference material
  • Data that changes often
  • Full examples or templates
  • Detailed how-to guides
  • Info only needed sometimes
›
3
Files
Reference documents Claude searches when relevant — not read every message
Deep reference Searched on demand 3–7 files ideal

Put here ✓

  • Reference guides & how-tos
  • Templates & example banks
  • Strategy & framework docs
  • Technical specs & API docs
  • Project registries

Not here ✗

  • Data that changes weekly+
  • Short facts (use memory)
  • Behavioral rules (use instructions)
  • Large datasets (use Notion)
  • Docs rarely needed
›
4
Notion
Living documents fetched on demand — the only layer that updates in real time
Dynamic data Tool call to fetch Always current

Put here ✓

  • Databases & trackers
  • Logs & decision history
  • Status dashboards
  • Cross-project shared pages
  • Anything "what's the latest?"

Not here ✗

  • Behavioral rules (use instructions)
  • Stable reference material (use files)
  • Personal preferences (use memory)
  • Detailed guides (use files)
